Introduction

This is the documentation for Team T.

Here you will find information about the API's endpoints, how to authenticate requests, and what to do if an error occurs.

Endpoints

Here is a list of the API's endpoints:

Method URL Request Body(JSON) Response(JSON) Error(e)
GET /api/users [user, user, ...] (1)
GET /api/users/{id}
id: A user ID		 user (1) (e1)
POST /api/users user(1) without id (e2)

Request Body and Response Formats

(1) User format (don’t provide ID, for POST)

    "{
    "age": Number,
    "name": String,
    "gender": String [“male” | “Female”],
    "email": String (email)
    }

Errors

If an error occurs while making a request to the API, an error response will be returned.

The error response will include an error code and message that can be used to diagnose the issue.

Here is a list of possible error codes:

  • (e1): 404 Not Found: No content found for this request
  • (e2): 400 Bad Request: Field ‘xxx’ is required
