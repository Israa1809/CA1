Request Body and Response Formats

(1a) Person format

    {
    "firstname": String,
    "lastname": String,
    "email": String (email)
    }

(1b) Person format

    {
    "person_id": int,
    "firstname": String,
    "lastname": String,
    "email": String (email),
    "Address": Address,
    "Hobbies": Set Hobby,
    "Phones": Set Phone
    }

(2a) Address format

    {
    "address_id": int,
    "street": String,
    "additionalInfo": String,
    "zipCode": ZipCode
    }

(2b) Address format

    {
    "address_id": int,
    "street": String,
    "additionalInfo": String,
    "zipCode": ZipCode,
    "Persons": Set Person
    }

(3) ZipCode format

    {
    "zipCode": int,
    "city": String
    }

(4a) Hobby format

    {
    "name": String,
    "wikiLink": String,
    "category": String,
    "type": String
    }

(4b) Hobby format

    {
    "name": String,
    "wikiLink": String,
    "category": String,
    "type": String,
    "Persons": Set Person
    }

(5a) Phone format

    {
    "number": int,
    "description": String
    }

(5b) Phone format

    {
    "number": int,
    "description": String,
    "Person": Person
    }

Errors

If an error occurs while making a request to the API, an error response will be returned.

The error response will include an error code and message that can be used to diagnose the issue.

Here is a list of possible error codes:

  • (e1) :{ status : 404, "msg": "No content found for this request" }
  • (e2) :{ status : 400, "msg": "Field ‘xxx’ is required" } (for example, no name provided)
  • (e3) :{ status : 500, "msg": "Server error"}
